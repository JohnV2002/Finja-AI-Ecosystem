⚡ Zero Data Retention — Das Wichtigste zuerst

Alle Text-basierten KI-Anfragen laufen über OpenRouter mit aktiviertem Zero Data Retention (ZDR).

  • OpenRouter leitet deine Nachricht an einen KI-Provider weiter (z.B. Google, Meta, Qwen).
  • Mit ZDR garantiert OpenRouter, dass der Provider deinen Text nicht zum Trainieren seiner Modelle benutzt.
  • Provider, die sich nicht an ZDR halten, werden von OpenRouter aktiv blockiert.
  • Dein Text wird verarbeitet, eine Antwort generiert, und dann verworfen. Kein Logging, kein Training, keine Speicherung beim Provider.
✅ Bild-Generierung jetzt auch mit ZDR — Seit Mai 2026 unterstützt das von YourAI genutzte Bildmodell (SeedDream) Zero Data Retention über OpenRouter. Der Bild-Prompt wird verarbeitet und dann verworfen — kein Logging, kein Training beim Provider.

7. Memory-System — Was YourAI über dich lernt

YourAI hat ein Langzeitgedächtnis. Gespeichert werden Fakten wie "User interessiert sich für Chemie" — nicht wörtliche Nachrichtentexte.

  1. Dein Text → LLM extrahiert Fakten (OpenRouter, ZDR, 🇺🇸)
  2. Fakten werden als Vektoren gespeichert (Memory-Server, 🇩🇪 lokal)

Das LLM sieht die extrahierten Fakten, nicht den Originaltext. Beispiel: ["User programmiert in Python", "User hat eine Katze"]

⚡ Zero Data Retention — The Most Important Thing First

All text-based AI requests run through OpenRouter with Zero Data Retention (ZDR) enabled.

  • OpenRouter forwards your message to an AI provider (e.g., Google, Meta, Qwen).
  • With ZDR, OpenRouter guarantees that the provider will not use your text for training.
  • Providers that don't comply with ZDR are actively blocked by OpenRouter.
  • Your text is processed, a response is generated, then discarded. No logging, no training, no storage at the provider.
✅ Image Generation now also with ZDR — Since May 2026, the image model used by YourAI (SeedDream) supports Zero Data Retention via OpenRouter. The image prompt is processed and then discarded — no logging, no training at the provider.

7–9. Memory, Diary & Emotional System

Memory: Stores extracted facts (e.g., "User likes Python") — not verbatim text. Locally in the memory server (🇩🇪). Extraction via OpenRouter (ZDR).

Diary: Stores truncated message content — beginning (~62%) and end (~38%) are kept, the middle is cut for long texts (max 900 chars per entry in prompts). Weekly summaries include top entries with previews (first 100 chars), tags, and date ranges. Viewable in the Dashboard (entry count). Local server, rotated weekly. Deletable via Session UUID.

Diary Search: Uses Cohere API (🇺🇸) for embedding and reranking. Training is disabled in Cohere account and API settings.

Promise Tracker / Mood: Stored in persona_state.json locally. Relevant excerpts flow into the LLM system prompt via OpenRouter (ZDR).

10. Feedback System — 👍 / 👎

👍 Thumbs up: Reinforces YourAI's behavior toward you personally — no effect on other users.

👎 Thumbs down: Penalizes the expert that responded (YourAI uses a Mixture-of-Experts system). 3× thumbs down from the same user → expert swapped for that user only. 3× from an Admin → expert swapped globally.

Stored locally in feedback_data.json: user ID, tracking ID, feedback value.

✍️ Writing Style Analysis

YourAI analyzes your writing style to communicate more naturally with you. The analysis uses two layers:

  • Local (Regex): Basic patterns (sentence length, emoji usage, formality) are detected directly on the server
  • AI Analysis (OpenRouter, ZDR): At message 10, 20, 30, 40, 70, 100, and every 100 messages thereafter, a deeper analysis is performed via an AI model. Your recent messages are sent to OpenRouter (ZDR active)

The resulting style profile is stored and included in YourAI's system prompt — it therefore leaves the server via OpenRouter (ZDR).

Viewable in the Dashboard under the Profile tab. Can be deleted by J. Apps on request. Stored in docker_data/user_styles/ (per user UUID).
